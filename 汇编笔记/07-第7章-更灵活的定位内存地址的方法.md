# 第7章 更灵活的定位内存地址的方法

## 本章主线

这一章是寻址方式的进阶，核心价值是：让你开始把内存中的数据看成数组、字符串、结构，而不是零散地址。

## 按小节过一遍

### 7.1 `and` 和 `or`

- `and` 用于按位与，常用于清零某些位、屏蔽某些位
- `or` 用于按位或，常用于置位某些位
- 这两条指令在字符处理里非常常见

### 7.2 ~ 7.4 ASCII、字符形式数据、大小写转换

- 汇编里可以直接用字符常量，本质仍会变成对应 ASCII 值
- 英文字母大小写之间只差一位：
- 大写转小写：把第 5 位设成 1
- 小写转大写：把第 5 位清成 0

例如：

```asm
or  al, 00100000b      ; 倾向于转小写
and al, 11011111b      ; 倾向于转大写
```

### 7.5 ~ 7.10 更复杂的寻址形式

- `[bx+idata]`：基址 + 位移
- `[bx+si]` / `[bx+di]`：基址 + 变址
- `[bx+si+idata]`：基址 + 变址 + 位移
- `SI/DI` 的加入，让你能更自然地表示：
- 数组下标
- 字符串当前位置
- 结构中某字段偏移
- 二维数据或复合偏移

## 本章最该记住的结论

- 灵活寻址的价值，不是“语法花样多”，而是能把内存看成更高级的数据结构。
- `BX/SI/DI` 组合起来，本质上就是早期的“基址 + 索引 + 偏移”。
- ASCII 不只是背码表，更要会用位操作快速处理字符。

## 最小例子

```asm
; 把字符串中的一个字符转成大写
and byte ptr [bx+si], 11011111b
```

如果 `BX` 指向字符串首地址，`SI` 指向当前字符下标，这就是很标准的字符串处理写法。

## 易错点

- 易错 1：把 `idata` 当成“段地址”，其实它只是偏移中的常量部分。
- 易错 2：大小写位运算直接乱用到任意字符。
- 这些技巧只对英文字母区间稳定成立。
- 易错 3：写复杂寻址后自己不再清楚“当前是第几个元素、第几个字段”。

## 线下赛/逆向视角

- 看到 `[bx+si+xx]` 时，要开始怀疑这里是不是数组、结构体、表项。
- 看到反复对某字节 `and/or` 固定位时，要想到可能在做字符规范化、权限位处理或状态位处理。
