# 第3章 寄存器（内存访问）

## 本章主线

第 2 章解决了“地址怎么算”，这一章继续解决“数据怎么在寄存器和内存之间流动”。同时，本章第一次系统引入栈。

## 按小节过一遍

### 3.1 内存中字的存储

- 8086 内存按字节编址。
- 一个字要占两个连续地址。
- 采用小端存储：
- 低位字节放在低地址
- 高位字节放在高地址

这条规则很关键，看内存 dump、调试器窗口、逆向结构体时都会用到。

### 3.2 ~ 3.5 `DS:[address]`、字传送、数据段

- 8086 访问内存时通常需要段地址和偏移地址。
- 当你写 `[bx]`、`[0]`、`[address]` 这类形式时，段地址默认通常来自 `DS`。
- 所以“读内存前先装 `DS`”是基本动作。
- `mov` 除了寄存器到寄存器，也可以在寄存器和内存之间传数据。
- 用 16 位寄存器时，可进行字传送。
- 数据段本质上只是“把某段内存当作数据区来看待”。

### 3.6 ~ 3.10 栈、CPU 提供的栈机制、`push/pop`、栈段

- 栈是后进先出结构。
- 8086 用 `SS:SP` 指向栈顶。
- `push` 入栈时先调整 `SP` 再写入。
- `pop` 出栈时先读出，再调整 `SP`。
- 8086 上基本以字为单位操作栈。
- 栈段和代码段、数据段一样，也是对一段内存的逻辑定义。

## 本章最该记住的结论

- 内存中字采用小端序。
- 默认数据访问很多时候依赖 `DS`。
- 栈不是独立硬件，而是 CPU 对内存的一种特殊使用方式。
- `push/pop` 是后面 `call/ret/int/iret` 的基础。

## 最小例子

```asm
mov ax, 1000h
mov ds, ax
mov bx, 0002h
mov ax, [bx]      ; 读取 DS:0002 的一个字

push ax
pop dx
```

如果 `SP` 原来是 `0010h`，执行 `push ax` 后会变成 `000Eh`。

## 易错点

- 易错 1：忘了初始化 `DS`。
- 易错 2：把内存中字的高低字节顺序看反。
- 易错 3：觉得 CPU 会自动检查栈是否越界。
- 8086 不会替你管这件事，栈段、栈顶是否合法要靠程序员自己保证。

## 复习抓手

- 见到 `[xxx]` 先问：默认段是谁？
- 见到栈操作先问：`SP` 怎么变？
- 见到字数据先问：低字节和高字节分别落在哪个地址？
