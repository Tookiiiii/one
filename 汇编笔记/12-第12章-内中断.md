# 第12章 内中断

## 本章主线

这一章开始把“控制流”从普通跳转扩展到“中断响应”。它解决的是：当 CPU 内部发生特殊事件时，程序如何被强制转去执行另一段代码。

## 按小节过一遍

### 12.1 内中断的产生

- 内中断来自 CPU 内部事件
- 典型如：
- 除法错误
- 单步执行
- 断点或其他内部异常条件

### 12.2 ~ 12.4 中断处理程序、中断向量表、中断过程

- 中断处理程序是处理某类中断事件的代码
- CPU 通过中断类型码去中断向量表里查入口地址
- 中断向量表项一项占 `4` 字节：
- 前 2 字节偏移地址
- 后 2 字节段地址
- 中断过程由硬件自动完成一部分关键动作，比如保存当前执行环境并跳转

### 12.5 `iret`

- `iret` 和 `ret` 的关键区别是：
- `iret` 不只恢复返回地址，还恢复标志寄存器
- 所以中断返回比普通子程序返回多一层上下文恢复

### 12.6 ~ 12.10 0号中断、安装、设置向量

- 书里用除法错误中断作为典型例子
- 处理流程通常包括：
- 准备中断例程代码
- 把它放到合适的内存位置
- 改写中断向量表项
- 让对应类型码指向新例程

### 12.11 单步中断

- 如果 `TF = 1`，CPU 每执行一条指令后都会触发单步中断
- 调试器就是利用这个机制工作的

### 12.12 响应中断的特殊情况

- 并不是任何时刻一发生中断 CPU 都立刻响应
- 某些特殊指令之后，CPU 会临时延后响应
- 书里特别提醒了和 `SS` 相关的情况

## 本章最该记住的结论

- 中断向量表是“中断类型码 -> 处理程序入口”的映射表。
- `iret` 是中断处理程序的标准返回方式。
- 中断不是普通 `jmp`，它是由 CPU 和硬件协作完成的特殊控制流切换。

## 最小例子

0 号中断向量表项的位置：

- 中断类型码 `0`
- 表项地址 `0 * 4 = 0`
- 所以偏移在 `0000:0000`
- 段地址在 `0000:0002`

## 易错点

- 易错 1：把 `ret` 当成中断返回
- 中断返回要用 `iret`
- 易错 2：改了中断向量但没把代码放到对应位置
- 易错 3：写 ISR 时忽略现场保护

## 线下赛/逆向视角

- 看到程序改写低地址中断向量表，要立刻想到是在安装或劫持中断处理程序
- 看到 `TF`、单步、断点时，要把它和调试机制联系起来
