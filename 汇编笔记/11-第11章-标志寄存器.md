# 第11章 标志寄存器

## 本章主线

这一章是读懂条件分支的关键。很多人学汇编只会背 `je/jne/jb/jl`，但不知道它们到底在看什么。真正该背的是标志位本身。

## 按小节过一遍

### 11.1 ~ 11.5 `ZF/PF/SF/CF/OF`

- `ZF`：结果是否为 0
- `PF`：结果低 8 位中 1 的个数是否为偶数
- `SF`：结果符号位
- `CF`：无符号运算中的进位或借位
- `OF`：有符号运算中的溢出

最常用的是 `ZF/CF/SF/OF`。

### 11.6 `adc`

- 带进位加法
- 本质：`目标 = 目标 + 源 + CF`
- 适合多精度加法

### 11.7 `sbb`

- 带借位减法
- 本质：`目标 = 目标 - 源 - CF`
- 适合多精度减法

### 11.8 `cmp`

- `cmp` 的本质就是减法，但不保存结果，只改标志位
- 后面的条件转移都是靠读这些标志位来判断比较结果

### 11.9 条件转移

- `je/jz` 主要看 `ZF`
- 无符号大小比较主要看 `CF` 和 `ZF`
- 有符号大小比较要结合 `SF` 和 `OF`

### 11.10 DF 和串传送指令

- `DF` 决定串操作时地址递增还是递减
- `DF = 0`：向前
- `DF = 1`：向后
- 这也是为什么串操作前常见 `cld`

### 11.11 ~ 11.12 `pushf/popf` 和 Debug 表示

- `pushf`：把标志寄存器压栈
- `popf`：从栈恢复标志寄存器
- 它们让标志寄存器能被显式保存/恢复

## 本章最该记住的结论

- 比较时不要只想着“数学上的大于小于”，要明确自己是在做有符号比较还是无符号比较。
- `CF` 主要服务于无符号语义。
- `OF/SF` 主要服务于有符号语义。
- 条件跳转读的不是“语言意义”，而是具体标志位。

## 最小例子

```asm
cmp ax, bx
je  equal
jb  below_unsigned
jl  less_signed
```

同样是“更小”，`jb` 和 `jl` 的语义并不相同。

## 易错点

- 易错 1：把 `CF` 当成“通用小于标志”
- 它更偏向无符号语义
- 易错 2：看到 `cmp` 以为结果被保存了
- 它只改标志位
- 易错 3：忘了串操作可能受 `DF` 影响

## 复习抓手

- 每看到条件跳转，先问它依赖哪些标志位
- 每看到比较，先问它是在按有符号还是无符号解释数据
