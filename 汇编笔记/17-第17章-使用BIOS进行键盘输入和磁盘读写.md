# 第17章 使用 BIOS 进行键盘输入和磁盘读写

## 本章主线

这一章把前面关于键盘中断、BIOS 服务、端口、外设访问的知识合到一起，开始真的“做输入、做读盘”。

## 按小节过一遍

### 17.1 `int 9` 对键盘输入的处理

- 键盘中断例程会把扫描码处理成可供程序使用的信息
- 一部分结果进入键盘缓冲区
- 一部分变成状态信息
- 这说明程序平时调用键盘服务时，依赖的其实是 BIOS 已经提前加工好的结果

### 17.2 `int 16h` 读取键盘缓冲区

- `int 16h` 是 BIOS 键盘服务接口
- 常见功能就是读取一个键盘输入
- 程序员通常不必自己直接处理扫描码，而是通过它拿已整理好的输入

### 17.3 字符串输入

- 从单字符输入走向字符串输入，就必须考虑更多交互细节：
- 回显
- 退格
- 结束条件
- 缓冲区边界
- 这其实已经很像一个迷你行编辑器了

### 17.4 `int 13h` 进行磁盘读写

- `int 13h` 是 BIOS 磁盘服务
- 书里重点讲了软盘 CHS 模式：
- 磁道
- 磁头/面
- 扇区
- 还要提供数据缓冲区 `ES:BX`
- 读写后通常要检查 `CF` 和返回状态

## 本章最该记住的结论

- 键盘输入可以走 BIOS 已整理好的缓冲区接口，不必总是直接看扫描码。
- 读盘写盘在 BIOS 层通常按扇区进行。
- 一旦涉及磁盘读写，地址、缓冲区、返回状态都必须非常明确。

## 最小例子

### 读取一个键盘输入

```asm
mov ah, 00h
int 16h
```

### 读一个扇区的典型骨架

```asm
mov ah, 02h
mov al, 01h
; 后续还要设置 CH、CL、DH、DL、ES:BX
int 13h
```

## 易错点

- 易错 1：把扇区号从 0 开始记
- CHS 下扇区号通常从 1 开始
- 易错 2：忘了 `ES:BX` 是数据缓冲区
- 易错 3：调用 BIOS 服务后不检查 `CF`
