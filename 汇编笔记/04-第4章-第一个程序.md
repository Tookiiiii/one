# 第4章 第一个程序

## 本章主线

这一章把前面的硬件和指令知识，正式落到“怎么写一个能被汇编、连接、装载、执行、跟踪的程序”上。它解决的是工具链和程序生命周期问题。

## 按小节过一遍

### 4.1 一个源程序从写出到执行的过程

- 汇编源程序不是写完就能跑。
- 典型流程是：
- 写 `.asm`
- 汇编成 `.obj`
- 连接成 `.exe`
- 由操作系统装载入内存
- 设置执行入口并开始执行

### 4.2 源程序结构

- `assume`：告诉汇编器你期望段寄存器和哪个逻辑段对应
- `segment/ends`：定义逻辑段
- 标号：标记入口或数据位置
- `end start`：告诉汇编器程序入口是谁

这些东西很多是给工具链看的，不是 CPU 直接执行的机器指令。

### 4.3 ~ 4.6 编辑、编译、连接

- `masm` 负责汇编，把助记符转成目标代码
- `link` 负责连接，生成可执行文件
- 初学时容易只盯着源程序本身，却忽略了目标文件、连接、装载这些阶段对程序最终形态的影响

### 4.7 ~ 4.9 EXE 执行、谁装载程序、Debug 跟踪

- 程序不会自己“蹦进内存里跑”，总要有别的程序装载它
- 在 DOS 下通常由操作系统负责把 EXE 装入内存、设置好运行环境并交出控制权
- Debug 很重要，因为你可以看到：
- 当前 `CS:IP`
- 当前寄存器值
- 当前内存内容
- 单步执行后的变化

## 本章最该记住的结论

- 汇编程序有“源程序视角”和“执行时内存视角”，不能混为一谈。
- `assume`、逻辑段、连接入口这些概念都是程序组织的一部分。
- Debug 是学习汇编最直接的观察窗口。

## 最小骨架

```asm
assume cs:code

code segment
start:
    mov ax, 4c00h
    int 21h
code ends

end start
```

这个程序的意义不在于功能，而在于它把“入口、段、退出方式”都交代清楚了。

## 易错点

- 易错 1：以为 `assume` 会自动给段寄存器赋值。
- 它不会，真正装载段地址还要靠你写指令。
- 易错 2：只会写程序，不会跟踪程序。
- 汇编里很多错误不单步跟根本看不出来。
- 易错 3：不分清源代码、目标文件、可执行文件三个阶段。

## 复习抓手

- 想一遍：源程序里哪些是给汇编器看的，哪些最后会变成机器码？
- 想一遍：程序到底是谁装入内存的？入口如何确定？
