# 第15章 外中断

## 本章主线

前面讲了内中断和软件中断，这一章补上第三块：来自外设的中断。键盘是书里最典型的例子。

## 按小节过一遍

### 15.1 接口芯片和端口

- 外设不会直接把数据塞进 CPU
- 中间一般有接口芯片、控制器、端口
- CPU 既通过端口访问这些接口逻辑，也通过中断及时得知事件发生

### 15.2 外中断信息

- 外设随时可能发生事件，比如按键、磁盘完成、时钟节拍
- CPU 不能一直忙等，所以要靠中断机制异步响应

### 15.3 PC 机键盘的处理过程

- 键盘按下后先产生扫描码
- 扫描码送入相关端口
- 触发对应中断
- BIOS 的键盘中断例程再把它进一步处理成缓冲区数据或状态信息

### 15.4 编写 `int 9` 中断例程

- 键盘相关的典型中断就是 `int 9`
- 你可以改写或扩展它的行为
- 但这类修改必须谨慎，因为你是在接管系统级输入通路

### 15.5 安装新的 `int 9` 例程

- 一般做法是保存原向量、安装新向量、必要时链回原处理程序
- 书里给的是“按某个键扩展屏幕效果”的教学例子

## 本章最该记住的结论

- 外中断是 CPU 响应外设事件的核心机制。
- 键盘输入不是“字符直接进程序”，中间经过扫描码、端口、中断、BIOS 处理这一整套流程。
- Hook 中断本质上是在改系统事件处理链。

## 最小流程图

```text
按键 -> 键盘控制器 -> 端口 -> 触发 int 9 -> BIOS/自定义中断例程 -> 缓冲区/状态字节
```

## 易错点

- 易错 1：把扫描码直接当 ASCII
- 易错 2：改 `int 9` 时不保护现场
- 易错 3：覆盖原逻辑后不考虑系统原有功能是否还能正常工作

## 线下赛/逆向视角

- 老程序里看到键盘相关逻辑，不要只找 `int 16h`
- 很多底层逻辑实际上藏在 `int 9`、端口 `60h`、键盘缓冲区这些位置
