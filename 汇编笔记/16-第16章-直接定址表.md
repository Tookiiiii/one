# 第16章 直接定址表

## 本章主线

这一章表面上在讲标号和查表，实际上是在教一种程序设计思想：把“判断逻辑”变成“查表逻辑”。

## 按小节过一遍

### 16.1 描述单元长度的标号

- 标号不仅能标示地址，还常和数据定义一起隐含了对象长度
- 这能让代码在访问数据时更清晰

### 16.2 在其他段中使用数据标号

- 数据标号不只属于代码段
- 只要段关系清楚，在其他段里一样可以用标号描述数据位置

### 16.3 直接定址表

- 这章的关键思想是查表
- 比如把一个十六进制数 `0~F` 映射成字符 `'0'~'F'`
- 与其写一堆判断，不如准备一张表，按索引直接取

### 16.4 程序入口地址的直接定址表

- 如果表里存的不是普通数据，而是子程序入口地址，那么就能实现分发器
- 这其实就是很原始的函数表/跳表思想

## 本章最该记住的结论

- 查表法能把复杂分支变成简单索引。
- 表里既可以放数据，也可以放地址。
- 一旦表里放的是地址，程序控制流就能被数据驱动。

## 最小例子

```asm
table db '0123456789ABCDEF'

mov bx, offset table
mov al, 0Ah
mov ah, 0
add bx, ax
mov al, [bx]
```

执行后，`AL` 取到字符 `'A'`。

## 易错点

- 易错 1：忘了表项大小，导致索引步长不对
- 易错 2：把“数据表”和“地址表”混为一谈
- 易错 3：间接跳转/调用时没想清楚目标到底是谁

## 线下赛/逆向视角

- 看到连续常量表、字符串表、地址表时，要主动想是不是：
- 十六进制映射表
- 菜单分发表
- 状态机跳表
- VM 指令分发表
