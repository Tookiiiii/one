# 第2章 寄存器

## 本章主线

这一章要解决两个核心问题：

1. 8086 有哪些关键寄存器，它们怎么存数据？
2. 8086 为什么要用“段地址:偏移地址”来访问内存？

理解这章以后，你才真正知道 CPU 是怎么从 `CS:IP` 开始跑程序的。

## 按小节过一遍

### 2.1 ~ 2.3 通用寄存器、字在寄存器中的存储、几条基本指令

- 8086 的 4 个常用通用寄存器是 `AX/BX/CX/DX`，都是 16 位。
- 每个都可以拆成高低 8 位，如 `AX = AH + AL`。
- 一个字 `word` 是 16 位，一个字节 `byte` 是 8 位。
- 8086 同时兼容字节操作和字操作，这也是很多指令既能处理 8 位也能处理 16 位数据的原因。

### 2.4 ~ 2.7 物理地址、16位结构、8086 的寻址方式

- 物理地址是内存单元在整个线性空间中的真实地址。
- 8086 是 16 位结构 CPU，但地址总线是 20 位，所以它能访问 `1MB` 空间。
- 问题在于：16 位寄存器装不下 20 位地址。
- 8086 的解决办法是：用段地址和偏移地址合成物理地址。
- 公式：`物理地址 = 段地址 * 16 + 偏移地址`

### 2.8 ~ 2.12 段的概念、段寄存器、CS/IP、修改控制流

- 段首先是一种逻辑划分方式，不是内存天然就切好了段。
- 8086 有 4 个段寄存器：`CS/DS/SS/ES`
- `CS` 存代码段地址，`IP` 存下一条指令的偏移地址。
- CPU 总是从 `CS:IP` 指向的位置取指。
- 也就是说，程序控制流的本质，就是不断改变 `IP`，有时还要改变 `CS`。
- `jmp`、`call`、`ret`、`int` 后面都围绕这个核心展开。

## 本章最该记住的结论

- 8086 访问内存不是直接给 20 位地址，而是“段地址 + 偏移地址”。
- 同一个物理地址可能对应多个不同的段:偏移表示。
- `CS:IP` 是理解程序执行流的总入口。
- 程序员不能像改普通寄存器那样随便 `mov cs, ...`，必须借助控制转移指令。

## 最小例子

已知 `CS = 1234h`，`IP = 5678h`，求 CPU 当前取指的物理地址：

- `1234h * 10h = 12340h`
- `12340h + 5678h = 179B8h`

所以当前从 `179B8h` 处取指。

## 常见误区

- 误区 1：把“段”理解成硬件强制切好的固定块。
- 更准确地说，它是 8086 为解决寻址问题引入的一种组织方式。
- 误区 2：只会背公式，不知道它为什么存在。
- 真正原因是：16 位寄存器要表达 20 位地址。
- 误区 3：记住了 `CS:IP`，但没意识到后面所有跳转和中断都在改它。

## 线下赛/逆向视角

- 看到汇编里的跳转，不要只看“跳到哪”，要看“是改 `IP` 还是改 `CS:IP`”。
- 看到段寄存器时，要开始形成“默认段”和“显式段覆盖”的意识。
