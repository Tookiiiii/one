# 第14章 端口

## 本章主线

前面一直在说内存和中断，这一章开始告诉你：CPU 访问外设时，不一定走普通内存读写，还会走端口 I/O。

## 按小节过一遍

### 14.1 端口的读写

- 端口是 CPU 和外设接口芯片交互的入口之一
- 访问端口不能用普通 `mov`
- 要用：
- `in`
- `out`
- 端口地址通过地址总线传送，PC 里常见范围是 `0 ~ 65535`

### 14.2 CMOS RAM 芯片

- CMOS RAM 里不只有配置数据，还带实时时钟相关信息
- 书里选它做例子，是因为它既有实际意义，又能体现“通过端口访问硬件”

### 14.3 `shl` 和 `shr`

- 左移、右移不只是位运算基础
- 后面处理 BCD、拆高低位、做时间数据变换时非常常用

### 14.4 CMOS 中的时间信息

- 年、月、日、时、分、秒都存放在 CMOS 的固定单元
- 读这些信息时通常先选寄存器索引，再通过数据端口读取

## 本章最该记住的结论

- 端口不是普通内存单元，访问方式不同。
- `in/out` 是 I/O 指令，不是内存搬运指令。
- CMOS/RTC 这类硬件信息是通过端口和接口芯片拿到的。

## 最小例子

```asm
mov al, 0
out 70h, al
in  al, 71h
```

这个过程的典型意义是：

- 向 `70h` 写索引
- 再从 `71h` 读对应数据

## 易错点

- 易错 1：把端口和内存混着理解
- 易错 2：知道 `70h/71h`，但不知道读出来可能是 BCD 编码
- 易错 3：不会用位移指令拆数据

## 线下赛/逆向视角

- 看到 `in/out` 就要想到：这是在直接和硬件接口说话
- 看到 `70h/71h`，很容易联想到 CMOS/RTC 访问
