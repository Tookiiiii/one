# 第1章 基础知识

## 本章主线

这章不是在讲“怎么写指令”，而是在搭建一个最底层的硬件视角：CPU、内存、总线、主板、接口卡、地址空间到底是什么关系。后面所有汇编知识如果脱离这个视角，就容易只剩死记硬背。

## 按小节过一遍

### 1.1 ~ 1.3 机器语言、汇编语言的产生、汇编语言的组成

- 机器语言是机器指令的集合，CPU 真正能执行的是机器码，不是助记符。
- 汇编语言的主体是汇编指令，作用是把难记的机器码写成便于人理解的形式。
- 汇编语言不只包括“有机器码的汇编指令”，还包括：
- 伪指令：给汇编器看的，比如 `db/dw/dd/segment/ends`
- 其他符号：比如标号、操作符、表达式等
- 这点很重要，因为初学者很容易把“源程序里出现的东西”全都当成 CPU 会执行的内容。

### 1.4 ~ 1.10 存储器、指令和数据、存储单元、CPU 对存储器的读写、三类总线

- 指令和数据在内存里没有天然区别，都是二进制信息。区别在于 CPU 如何解释它。
- 内存由若干存储单元组成，按字节编址。
- CPU 读写内存至少要解决三件事：
- 找到地址：靠地址总线
- 传送数据：靠数据总线
- 发出读写控制：靠控制总线
- 地址总线宽度决定可寻址范围，数据总线宽度决定一次搬运多少数据，控制总线宽度决定 CPU 能对外部器件发出哪些控制能力。

### 1.11 ~ 1.15 内存地址空间、主板、接口卡、存储器芯片

- 从 CPU 的角度看，只要通过总线能统一访问到，就可以视为“内存地址空间”的一部分。
- 主板上不只有 RAM，还会有 ROM、接口芯片、显存映射、扩展卡等。
- CPU 不能直接控制显示器、键盘、磁盘这类外设，通常是通过接口卡或接口芯片间接控制。
- 这也是为什么后面会讲“端口”和“中断”，因为 CPU 和外设之间通常隔着接口逻辑。

## 本章最该记住的结论

- 汇编语言是对机器语言的可读化，不是另一套脱离硬件的抽象语言。
- 程序运行的本质，是 CPU 不断从内存取指、解释、执行。
- CPU 看外部世界，主要靠总线系统。
- 存储器不只是 RAM，整个系统中的很多设备都可能被 CPU 统一编址或统一控制。

## 一个最小思考题

如果 CPU 有 10 根地址总线，那么它能寻址多少个存储单元？

- 每根线可表示 0/1 两种状态
- 10 根线可表示 `2^10 = 1024` 种地址
- 所以可寻址 `1024` 个单元

## 常见误区

- 误区 1：觉得“数据”和“代码”在机器层面本质不同。
- 实际上它们都只是比特流，区别来自解释方式。
- 误区 2：把“地址空间”只理解成内存条的 RAM。
- 实际上很多硬件资源都可能映射到 CPU 可访问空间中。
- 误区 3：背总线定义，但不知道后面有何用。
- 后面你看到端口、显存、CMOS、中断控制时，都会回到这一章。

## 复习时的抓手

- 一看到“总线”就问：这是地址、数据还是控制？
- 一看到“设备”就问：CPU 直接管，还是通过接口芯片管？
- 一看到“内存空间”就问：这块空间到底映射了什么？
