# 第5章 [BX]和 loop 指令

## 本章主线

这一章把“怎么访问一串连续数据”和“怎么形成最基本的循环”结合起来，是从单条指令迈向程序控制的关键一步。

## 按小节过一遍

### 5.1 `[BX]`

- `[bx]` 表示以内存间接方式访问内存。
- 偏移地址来自 `BX`，段地址默认来自 `DS`。
- 所以 `[bx]` 的本质是访问 `DS:BX`。

这比写死 `[idata]` 更灵活，因为 `BX` 可以变。

### 5.2 ~ 5.5 `loop` 指令、Debug 跟踪、和 `[bx]` 的联用

- `loop label` 的动作分两步：
- `CX = CX - 1`
- 如果 `CX != 0`，跳到 `label`
- 因为 `BX` 能走地址，`CX` 能做计数，所以两者天然适合配合处理数组、字符串、批量内存单元。
- 这其实是最原始的“指针 + 计数器”思想。

### 5.4 Debug 和 masm 的差异

- 本章有一个很重要的提醒：Debug 里的输入方式和汇编源程序的语法处理并不完全一样。
- 尤其是像 `[0]` 这样的写法，在不同环境里含义和汇编器处理方式要格外小心。

### 5.6 ~ 5.8 段前缀、安全空间、段前缀的使用

- 默认访问段虽然常是 `DS`，但你可以显式写段前缀，如 `es:[bx]`
- 这让跨段拷贝、访问不同区域数据成为可能
- 但实模式下不能乱写内存，必须确认目标地址是安全空间

## 本章最该记住的结论

- `[bx]` 是最基本、最常用的间接寻址形式之一。
- `loop` 本质就是 `CX` 驱动的短跳循环。
- 处理一段连续数据时，经常就是 `BX`/`SI` 走地址，`CX` 管次数。
- 段前缀让你跳出默认段寄存器的限制。

## 最小例子

```asm
mov ax, data
mov ds, ax
mov bx, 0
mov cx, 8
xor al, al

s:
    add al, [bx]
    inc bx
    loop s
```

这个结构几乎就是后面所有“遍历数据”的原型。

## 易错点

- 易错 1：`loop` 不是“先判断再减”，而是“先减再判断”。
- 易错 2：`loop` 是短跳，范围有限。
- 易错 3：忘记 `[bx]` 默认走 `DS`。
- 易错 4：在不安全内存区域测试拷贝或写入。

## 线下赛/逆向视角

- 逆向时看到 `mov al, [bx] / inc bx / loop xxx`，第一反应就是“这是在扫一段连续字节”。
- 看到段前缀时，说明代码可能在同时处理多个逻辑区域的数据。
