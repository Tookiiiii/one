## 这本书的真正主线

可以把全书压成下面几句话：

1. CPU 从 `CS:IP` 指向的位置取指。
2. 访问内存时，8086 用 `段地址 << 4 + 偏移地址` 生成物理地址。
3. 数据可以在寄存器、内存、立即数里，关键是“指令如何找到它”。
4. 栈靠 `SS:SP` 工作，`push/pop/call/ret/int/iret` 都和栈强相关。
5. 程序控制流的改变，本质就是改 `IP`，进一步就是改 `CS:IP`。
6. 条件跳转的依据不是“高级语义”，而是标志寄存器中的位。
7. 中断机制让 CPU 能响应内部事件、外部设备和软件请求。
8. BIOS、DOS、中断向量表、端口、CMOS、键盘缓冲区这些东西连起来，才是“8086 程序运行在真实机器上”的完整图景。

## 必背公式和事实

- 物理地址 `= 段地址 * 16 + 偏移地址`
- 一个段最大 `64KB`
- 中断向量表项大小 `4` 字节
- 中断向量表项地址 `= 中断类型码 * 4`
- `BX/SI/DI` 默认段寄存器是 `DS`
- `BP` 默认段寄存器是 `SS`
- `ret` 弹 `IP`
- `retf` 弹 `IP + CS`
- `iret` 弹 `IP + CS + FLAGS`
- `loop label` 等价于：`CX = CX - 1`，若 `CX != 0` 则跳
- `cmp a, b` 只做比较，不保存结果，本质是 `a - b`

## 二刷时每章都要问自己的问题

- 这一章主要在解释“CPU 怎么看世界”的哪一部分？
- 这里的地址，是物理地址、段地址还是偏移地址？
- 这里的跳转，是段内还是段间？
- 这里是不是在隐式用栈？
- 这里读的标志位到底是 `CF/ZF` 还是 `SF/OF`？
- 这里访问的是内存、端口、BIOS 服务，还是 DOS 服务？
