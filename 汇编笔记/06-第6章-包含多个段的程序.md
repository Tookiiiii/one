# 第6章 包含多个段的程序

## 本章主线

这一章的重点不是新指令，而是“程序怎么组织更像一个程序”。前面默认把东西都塞进一起能跑，但从这里开始，书里强调数据、代码、栈分离的意义。

## 按小节过一遍

### 6.1 在代码段中使用数据

- 技术上完全可以在代码段里放数据。
- 因为内存里本来没有“这里天生是代码、那里天生是数据”的区别。
- 但这样做通常只是教学或特定技巧需要，不利于结构清晰。

### 6.2 在代码段中使用栈

- 同理，代码段中也能临时拿一部分空间当栈。
- 关键不在“能不能”，而在你是否清楚当前 `SS:SP` 指向哪里。

### 6.3 将数据、代码、栈放入不同的段

- 真正有组织的程序，通常会把：
- 代码放代码段
- 数据放数据段
- 栈放栈段
- 然后显式初始化 `DS`、`SS`、`SP`
- 这会让程序逻辑、调试和维护都清楚很多

## 本章最该记住的结论

- 逻辑段的划分是一种程序组织思想。
- `assume` 只是告诉汇编器你准备怎么用这些段，不是自动装载段地址。
- 多段程序写起来麻烦一点，但换来的是更清晰的边界。

## 最小骨架

```asm
assume cs:code, ds:data, ss:stack

data segment
    arr db 1,2,3,4
data ends

stack segment
    db 32 dup (0)
stack ends

code segment
start:
    mov ax, data
    mov ds, ax

    mov ax, stack
    mov ss, ax
    mov sp, 32

    mov ax, 4c00h
    int 21h
code ends

end start
```

## 易错点

- 易错 1：写了 `assume ds:data`，就误以为 `DS` 已经等于 `data`
- 易错 2：初始化了 `SS` 却没初始化 `SP`
- 易错 3：把数据、栈、代码混用后，调试时自己都分不清当前在访问什么

## 复习抓手

- 每次看到一个段，就问：它的职责是什么？
- 每次看到内存访问，就问：这次访问是站在代码段、数据段还是栈段的语境里？
