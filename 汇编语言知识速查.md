# 汇编语言知识速查（8086 / 实模式 / 线下赛向）

> 基于《汇编语言（第4版）王爽》与常见 CTF/逆向/引导题经验整理。  
> 目标：不是“背指令大全”，而是建立**机器级思维**与**临场速查能力**。

---

## 0. 使用说明（先看这一页）

### 0.1 默认语境

- CPU：`8086/8088` 风格
- 模式：`Real Mode（实模式）`
- 系统接口：`BIOS + DOS int 21h`
- 地址写法：`段地址:偏移地址`（如 `07C0:0000`）

### 0.2 主线只记一条

`分段 -> 寻址 -> 栈 -> 转移 -> 子程序 -> 标志位 -> 中断 -> I/O`

线下赛里绝大多数汇编题，本质都绕不开这条链路。

### 0.3 最重要的学习目标

不要只记“这条指令怎么写”，而要看懂：

1. CPU 下一条指令从哪取（`CS:IP`）
2. 内存地址怎么算（段基址+偏移）
3. 控制流何时改变（`jmp/call/ret/int/iret`）
4. 状态如何影响分支（`FLAGS`）
5. 如何与硬件交互（中断与端口）

## 1. 必背公式与核心事实

### 1.1 地址与段

- **物理地址** = `段地址 << 4 + 偏移地址`
- 单段最大大小：`64KB`
- 同一物理地址可由多组段:偏移表示（有重叠映射）

示例：

```asm
1234:5678 -> 12340h + 5678h = 179B8h
```

### 1.2 栈与控制流

- 栈顶默认由 `SS:SP` 指向，栈向低地址增长
- `call`：压栈返回地址后跳转
- `ret`：弹出 `IP`
- `retf`：弹出 `IP` 和 `CS`
- `iret`：弹出 `IP`、`CS`、`FLAGS`

### 1.3 中断向量表（IVT）

- 位于物理地址 `00000h~003FFh`
- 共 `256` 个向量，每个向量 `4` 字节：`IP(2) + CS(2)`
- 第 `n` 号中断向量地址：`n * 4`

### 1.4 几条常见“本质理解”

- `cmp a, b` 本质是 `a-b`，只改标志位，不保存结果
- `test a, b` 本质是按位与，仅改标志位
- `loop label` 本质：`CX = CX - 1`，若 `CX != 0` 则短跳

## 2. 寄存器速懂（含实战关注点）

### 2.1 通用寄存器

- `AX`：累加器，很多算术/乘除默认使用
- `BX`：基址寄存器，常参与内存寻址
- `CX`：计数器，`loop/rep` 常用
- `DX`：数据寄存器，乘除和端口 I/O 常用

可拆分高低 8 位：`AH/AL`、`BH/BL`、`CH/CL`、`DH/DL`

### 2.2 指针与变址

- `SP`：栈顶指针（配 `SS`）
- `BP`：基址指针（默认段寄存器是 `SS`）
- `SI/DI`：变址寄存器（字符串与数组处理高频）

### 2.3 段寄存器与指令指针

- `CS`：代码段
- `DS`：数据段
- `SS`：栈段
- `ES`：附加段
- `IP`：下一条待执行指令偏移

> 重点：普通 `mov` 不能直接写 `CS`，修改 `CS:IP` 依赖远跳转/远调用/中断返回等机制。

---

## 3. 寻址方式（最容易丢分）

### 3.1 默认段寄存器规则

- 用到 `BP`：默认段是 `SS`
- 其他常见（`BX/SI/DI`）：默认段是 `DS`

### 3.2 常见写法

```asm
[idata]           ; 直接寻址
[bx] [si] [di]    ; 寄存器间接
[bp]              ; 寄存器间接（默认SS）
[bx+idata]
[bp+idata]
[bx+si] [bx+di] [bp+si] [bp+di]
[bx+si+idata]
```

### 3.3 段超越前缀（segment override）

需要强行改默认段时可写：

```asm
mov ax, es:[di]
mov [es:di], ax
```

逆向里这是判断数据来源的重要线索。

---

## 4. 内存、字节序、数据宽度

### 4.1 小端序（Little Endian）

一个字（16 位）在内存中低字节在低地址：

```text
1234h 存储为：34h 12h
```

### 4.2 宽度匹配

- `byte ptr` / `word ptr` 用于明确操作宽度
- 某些场景不写会导致歧义或汇编失败

```asm
mov byte ptr [bx], 12h
mov word ptr [bx], 1234h
```

### 4.3 指令与数据没有“天然边界”

在机器层，它们都是比特流。是否被当作代码执行，取决于 CPU 当前是否把该地址当作取指位置。

> 这解释了很多“代码段藏数据 / 数据段藏代码”的逆向技巧。

---

## 5. 标志位 FLAGS（分支判断核心）

### 5.1 高频标志

- `CF`：进位/借位（无符号语义）
- `ZF`：结果是否为 0
- `SF`：结果符号位
- `OF`：有符号溢出
- `PF`：低 8 位中 1 的个数是否为偶数
- `AF`：BCD 调整相关（一般了解）

### 5.2 无符号 vs 有符号（高频混淆点）

- 无符号比较主要看：`CF/ZF`
- 有符号比较主要看：`SF/OF/ZF`

典型映射：

- `ja`：无符号大于（`CF=0 && ZF=0`）
- `jb`：无符号小于（`CF=1`）
- `jg`：有符号大于（`ZF=0 && SF=OF`）
- `jl`：有符号小于（`SF!=OF`）

---

## 6. 转移指令与子程序

### 6.1 跳转

- `jmp short`：短跳（相对位移 8 位）
- `jmp near`：段内跳
- `jmp far`：段间跳（改 `CS:IP`）

### 6.2 调用

- `call near`：压栈返回 `IP`
- `call far`：压栈返回 `CS:IP`

### 6.3 过程约定（读反汇编很有用）

常见函数序言/尾声：

```asm
push bp
mov  bp, sp
sub  sp, N
...
mov  sp, bp
pop  bp
ret
```

参数常见通过 `[bp+4] [bp+6] ...` 访问（具体与调用约定有关）。

---

## 7. 字符串指令与方向标志 DF

### 7.1 常见指令

- `movsb/movsw`：搬运
- `cmpsb/cmpsw`：比较
- `scasb/scasw`：扫描
- `lodsb/lodsw`：装载到 `AL/AX`
- `stosb/stosw`：存储 `AL/AX`

### 7.2 与 `rep` 搭配

- `rep movsb`：复制 `CX` 个字节
- `repe/repz cmpsb`：相等则继续
- `repne/repnz scasb`：不等则继续

### 7.3 DF 影响方向

- `cld`：`DF=0`，`SI/DI` 递增（正向）
- `std`：`DF=1`，`SI/DI` 递减（反向）

---

## 8. 中断机制（考试+实战双高频）

### 8.1 中断执行过程（简化）

1. 压栈 `FLAGS`
2. 压栈 `CS`
3. 压栈 `IP`
4. 清 `TF/IF`（典型情形）
5. 跳到向量表指定的 `CS:IP`

返回用 `iret`。

### 8.2 BIOS / DOS 常见中断

- `int 10h`：显示服务（BIOS）
- `int 13h`：磁盘服务（BIOS）
- `int 16h`：键盘服务（BIOS）
- `int 19h`：引导装载相关
- `int 21h`：DOS 功能调用

高频 DOS 功能：

- `int 21h, AH=09h`：输出 `$` 结尾字符串
- `int 21h, AH=4Ch`：返回 DOS

---

## 9. 端口 I/O（硬件交互入口）

### 9.1 基本指令

- `in al, dx` / `in ax, dx`
- `out dx, al` / `out dx, ax`

### 9.2 常见应用

- 键盘控制器、PIC、PIT、CMOS、VGA 等都可通过端口访问
- 逆向固件/Bootloader 时常见“端口轮询 + 状态位判断”

## 10. Debug 调试速查（DOS DEBUG 思路同样适用于现代调试器）

| 命令 | 作用 |
| --- | --- |
| `r` | 查看/修改寄存器 |
| `d` | 查看内存 |
| `e` | 修改内存 |
| `u` | 反汇编 |
| `a` | 汇编输入 |
| `t` | 单步跟踪（step into） |
| `p` | 过程步过（step over） |
| `g` | 运行到断点/结束 |
| `q` | 退出 |

调试优先观察四件事：

1. 它在访问哪里（地址）
2. 它在改哪些寄存器
3. 它是否改变控制流
4. 它依赖哪些标志位

## 11. 线下赛 / CTF 实战切题建议

### 11.1 时间不够时先背这些

- 分段与物理地址换算
- 默认段寄存器规则（尤其 `BP -> SS`）
- 栈与 `call/ret/iret`
- `cmp/test + jcc` 条件体系
- IVT 结构与常见 BIOS/DOS 中断

### 11.2 逆向题优先关注

- 函数边界（序言/尾声）
- 参数位置（栈或寄存器）
- 分支条件真实语义（有符号/无符号）
- “混淆式控制流”（跳转链、表跳转、中断跳转）

### 11.3 引导区题优先关注

- 加载地址与入口（常见 `0000:7C00`）
- `int 13h` 读盘参数是否正确
- 段寄存器初始化是否完整
- 栈是否已设置（未设栈极易炸）

## 最后一句

真正拉开差距的，不是你背了多少指令，而是你能否在 10 秒内回答：

- 这条指令改了什么状态？
- 下一条指令会从哪执行？
- 这个条件跳转到底按有符号还是无符号解释？

能稳定回答这三问，8086 实模式题基本都能落地。
